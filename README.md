# Vanishing Punctuated Dynamics on High-Dimensional Fitness Landscapes

Monte Carlo simulation code for the paper:

> [**Vanishing punctuated dynamics on high-dimensional fitness landscapes: a deep-basin condition**](https://doi.org/10.5281/zenodo.19091703)
>
> [Hiroyuki Nashida](https://orcid.org/0009-0009-2602-4549) (2026).

## Overview

This repository provides the simulation and analysis code for studying evolutionary dynamics on NK fitness landscapes with tunable fitness basins. The simulations discover three dynamical regimes (diffusive, punctuated, frozen) and demonstrate that the punctuated regime vanishes as genetic dimensionality increases beyond d ≈ 20.

### Key Findings

- **Three regimes**: Diffusive (low β), punctuated (intermediate β), frozen (high β)
- **Vanishing punctuated band**: Present at d = 10–18, absent at d ≥ 20
- **Deep-basin condition**: β*_low(d) < β < β*_high(d) required for punctuated dynamics
- **d_eff upper bound**: Landscape-geometry-mediated punctuated equilibrium requires d_eff ≲ 20

## Quick Start

### Installation

```bash
git clone https://github.com/harappa/nk-punctuated-band.git
cd nk-punctuated-band

python3 -m venv .venv
source .venv/bin/activate    # Linux/macOS
# .venv\Scripts\activate     # Windows

pip install -r requirements.txt
```

### Run All Experiments

```bash
# Full run (~40 min on 16 cores; ~23,000 MC chains, >10^11 total steps)
python scripts/run_experiments.py

# Run specific experiments
python scripts/run_experiments.py --exp 1 2 3

# Quick validation mode (short runs for testing)
python scripts/run_experiments.py --quick

# Override worker count
python scripts/run_experiments.py --workers 16
```

### Self-Test

```bash
python scripts/self_test.py
```

This runs:
1. Compilation check (Numba JIT warmup)
2. Regime verification (known diffusive/punctuated/frozen points)
3. Performance benchmark across dimensions

## Reproducibility

All simulation results reported in the paper can be reproduced from this code. Every Monte Carlo chain uses a **deterministic random seed** derived from a SHA-256 hash of its parameter tuple (dimension, K, β, s_coeff, h*, replicate index). The seed fully determines both the NK fitness landscape and the entire sequence of random numbers consumed during the simulation.

Specifically:

- **Exp 1–7** (single-lineage Metropolis): All random numbers are pre-generated by `numpy.random.RandomState` (MT19937) with the deterministic seed, then passed into the Numba JIT-compiled core. No random numbers are generated inside the JIT function.
- **Exp 8** (polynomial basins): Same design as Exp 1–7.
- **Exp 9** (Wright-Fisher): Random numbers for mutation and selection are generated inside the JIT function via Numba's internal `np.random`, seeded deterministically at the start of each run.

**Conditions for bit-exact reproduction:**

Running on the same CPU architecture with the same versions of Python, NumPy, and Numba will produce bit-identical results. `numpy.random.RandomState` (MT19937) is a standardized PRNG whose output sequence is fixed for a given seed across all platforms sharing the same NumPy version.

**When statistical (but not bit-exact) reproduction is expected:**

Different CPU architectures (e.g., x86-64 vs. ARM) or different library versions may introduce minor floating-point rounding differences due to SIMD instruction sets and compiler optimizations. These differences are negligible and do not affect regime classification or any statistical conclusion reported in the paper. This is a standard limitation in computational science.

## Model

### Fitness Function

Each genotype **g** ∈ {0,1}^d has fitness:

```
w(g) = NK_component(g) + β · max(exp(-2ḡ²), exp(-2(1-ḡ)²))
```

where ḡ = d⁻¹ Σᵢ gᵢ is the Hamming fraction, and the NK component provides epistatic ruggedness with K nearest-neighbor interactions.

### Evolutionary Dynamics

Single-lineage Metropolis dynamics in the weak-mutation limit (Nμ ≪ 1):
- One random locus flipped per step
- Acceptance probability: min(1, exp(s_coeff · Δw))
- Genotypes classified as basin 1 (ḡ < h*), basin 2 (ḡ > 1−h*), or transitional

### Regime Classification

- **Diffusive**: stasis/transit ratio < 5
- **Marginal**: 5 ≤ ratio < 10
- **Punctuated**: ratio ≥ 10
- **Frozen**: mean n_transitions < 1 across replicates
